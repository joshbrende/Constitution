## Data Readiness and AI

### Data-as-a-Service (DaaS) — Concepts

**Data-as-a-Service** (DaaS) is a way of thinking about data: treat it as a **managed asset** that is **discoverable**, **accessible** (to those authorised), and **reliable** for different uses—including AI and analytics. In practice, this often means:

- **Cataloging:** A list or catalog of what data exists, where it is, who owns it, and how to access it.
- **Standard interfaces:** Consistent ways to access data (APIs, exports, secure links) rather than ad hoc copies and manual handovers.
- **Quality and governance:** Clear ownership, rules, and checks so that consumers (including AI) can trust the data.

You do not need a full DaaS platform to start. A **data inventory** (Practical section) and **data–KPI mapping** are first steps. **SALGA Outcome 2 (Customised Innovative Solutions)** includes big data management; improving data readiness supports that.

[STEP] Getting “AI-ready”

## Getting “AI-Ready”

Data is **AI-ready** when it is in a form and quality that supports your intended AI use:

| Use | What “ready” often means |
|-----|---------------------------|
| **Structured analytics (dashboards, KPIs)** | Data in consistent format; clear definitions; regular updates; linkable to your KPI definitions. |
| **Document AI (summaries, extraction from IDP/SDBIP)** | PDFs or text in a consistent structure; clear versioning; no confidential personal data if using public AI. |
| **Predictive models (e.g. collection, demand)** | Historical data with enough records; key fields (date, amount, category) complete and valid; definition of the target (e.g. “collected within 90 days”) consistent over time. |
| **Sentiment or feedback analysis** | Text in digital form; anonymised or aggregated if using public AI; clear scope (e.g. which channels, which period). |

[STEP] Prioritisation — what to improve first

## Prioritisation: What to Improve First

You cannot fix all data at once. Prioritise by:

1. **Impact on KPIs and audits.** Data that feeds **SDBIP, annual report, CoGTA, or SALGA** comes first.
2. **Impact on high-value decisions.** Revenue, critical service delivery, and compliance often rank high.
3. **Feasibility.** Where is quality already reasonable? Where is the owner engaged? Start there to show quick wins.
4. **AI and dashboard roadmap.** Align with Modules 5–12: if you will build a revenue dashboard (Module 6), revenue data is a priority.

The **Practical** section’s **data source–KPI mapping** helps you see which data supports which KPIs and where the gaps are. Use that to prioritise.
